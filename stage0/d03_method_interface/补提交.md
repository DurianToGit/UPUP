一、方法集
1. T和*T的方法集分别包含什么？
T：仅包含值接收者的方法
*T: 包含值接收者和指针接收者的方法
2. 为什么d.Save()能够调用成功，
   但var s Saver = d可能失败？
回答：因为d的Save方法可能是指针接收者，而d可能只是值，不是指针值，所以var s Saver = d可能失败
3. 接口实现判断时，编译器是否会自动对值取地址？
回答：不一定；
如果参数为指针，实现方法为值接收者，则编译器会自动对值取地址
如果参数非指针值，实现方法为指针接收者，则编译器不会对值取地址，且会编译报错
4. 普通函数怎样通过命名函数类型实现接口？
回答：
type FuncName func(params string) error

func (f FuncName) Notify(params string) error {
   return f(params)
}
6. 接口应该定义在哪里？为什么不能说绝对由调用方定义？
回答：优先考虑接口的调用方，因为某些特定的场景下，也会定义到实现方
7. Typed Nil为什么不等于nil？
回答：因为判断接口是否为nil,需要判断，type和value都为nil,而typed Nil的type不为nil
